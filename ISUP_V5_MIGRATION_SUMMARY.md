# 海康 ISUP v5.0 协议改造完成总结

## 改造项目概述

已成功将项目从 ISUP v3.0 升级到 v5.0，支持设备上报报文的正确解密解包。

## 核心修改内容

### 1. IsupMessageHeader.java - 消息头结构升级

**关键变更：**
- 消息头长度：32 字节 → **92 字节**
- 设备 ID 长度：8 字节 → **48 字节**（源和目标各 48 字节）
- 新增 IV 向量字段：**16 字节**（用于 AES-CBC 模式）
- 保留位扩展：3 字节 → **12 字节**
- 协议版本支持：新增 **0x05 (ISUP v5.0)**

**消息头结构（92 字节）：**
```
| 字段          | 偏移量 | 长度 | 说明                    |
|---------------|--------|------|-------------------------|
| 起始标识      | 0      | 2    | 固定 0x687A             |
| 消息长度      | 2      | 4    | 整个消息长度（大端）     |
| 协议版本      | 6      | 1    | 0x05 = ISUP v5.0        |
| 保留位        | 7      | 1    | 固定 0x00               |
| 消息类型      | 8      | 2    | 消息类型代码            |
| 序列号        | 10     | 4    | 消息唯一标识            |
| 源设备 ID     | 14     | 48   | 设备序列号（ASCII）      |
| 目标设备 ID   | 62     | 48   | 平台 ID（ASCII）         |
| 加密标志      | 110    | 1    | 0x00=不加密，0x01=AES-CBC|
| IV 向量       | 111    | 16   | CBC 模式初始化向量        |
| 保留位 2      | 127    | 12   | 固定 0x00               |
| CRC 校验码    | 139    | 2    | CRC16 校验              |
```

### 2. IsupEncryptionUtil.java - 加密工具类升级

**关键变更：**
- 加密模式：AES/ECB/PKCS5Padding → **AES/CBC/PKCS5Padding**
- 新增 IV 向量参数支持
- 新增 `generateKeyFromDeviceCode()` 方法（从设备验证码生成密钥）
- 新增 `generateRandomIv()` 方法（生成随机 IV）

**API 变更：**
```java
// 旧 API（v3.0）
decrypt(byte[] data, byte[] key, byte encryptionFlag)

// 新 API（v5.0）
decrypt(byte[] encryptedData, byte[] key, byte[] iv)
encrypt(byte[] plainData, byte[] key, byte[] iv)
```

### 3. IsupServerHandler.java - 协议处理器升级

**关键变更：**
- 最小消息长度：32 字节 → **92 字节**
- 消息头解析：使用新的 92 字节结构
- 解密流程：从消息头提取 IV 向量，使用 AES-CBC 模式
- 密钥管理：注册时自动提取设备验证码并生成会话密钥
- 调试增强：添加详细的日志输出和消息头打印

**解密流程：**
1. 从消息头提取 16 字节 IV 向量
2. 从会话管理器获取设备密钥（或从设备验证码生成）
3. 使用 AES-128-CBC 模式解密消息体
4. 注册请求成功后保存设备密钥供后续使用

## 密钥管理机制

1. **初始注册**：使用默认密钥或配置密钥解密注册请求
2. **提取验证码**：从注册请求体中提取 48 字节设备验证码
3. **生成会话密钥**：取设备验证码前 16 字节作为 AES 密钥
4. **后续通信**：使用会话密钥解密该设备的所有加密消息

## 配置建议

在 `application.yml` 中添加以下配置：

```yaml
isup:
  server:
    port: 8700  # ISUP 服务端口
  encryption:
    enabled: true
    default-key: "hik12345"  # 默认密钥（仅用于初始注册）
  device:
    timeout: 300  # 设备超时时间（秒）
```

## 问题排查指南

### 解密失败常见原因

1. **密钥不匹配**
   - 检查设备验证码是否正确配置
   - 确认密钥生成算法一致

2. **IV 向量错误**
   - 确保从消息头正确提取 16 字节 IV
   - IV 必须与加密时使用的一致

3. **设备 ID 长度问题**
   - v5.0 必须使用 48 字节设备 ID
   - 检查序列化/反序列化逻辑

4. **加密模式错误**
   - 必须使用 CBC 模式，不是 ECB
   - 确保 Java Cryptography Extension 支持
